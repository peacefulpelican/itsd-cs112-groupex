CS112 Group Assessment Specification 2026
This assessment will be tackled as a group. Only one submission will be required for the
whole group. You will be marked out of 10 and the submission is worth 4% of your overall
grade. Submissions must be uploaded to MyPlace by 17:00 on Monday 6th April 2026.
Overview
In groups of four, you will create a small Java console program to manage book records.
The program must allow the user to:
•
•
•
•
Add a book record
Display all book records
Save records to a file
Load records from a file
You will be provided with an interface. Each group member should primarily work on one
class. If your group only has 3 members, you do not need to complete section 4 Main Class.
Provided Interface
import java.io.Serializable;
public interface DisplayableRecord extends Serializable {
String getId();
void generateID()
void getDisplayText();
}
This interface represents an object that:
•
•
•
can be uniquely identified
can produce text suitable for displaying to a user
can be serialised to a file

author is empty
year is less than 0 or greater than 2026
It should also generate the id for the book using the method generateID(). The id should

characters of the author name. For example, the book with title “The Hobbit” by author
“J.R.R Tolkien” published in year 1937 will have the id “The1937J.R”

Purpose
Contains the main program and menu.
Required Method
public static void main(String[] args)
The main method should:
•Create a Scanner for reading user input.
•Create a RecordManager instance.
•Create a FileHandler instance using the appropriate filename.
•Display a menu repeatedly until the user selects the option to exit.
•Read and validate the user's menu choice. Display error message for invalid
choices.
•Invoke the appropriate actions based on the user’s choice.
•Handle any exceptions that may occur during user input or program execution.
To keep the main method clear and focused, you should design this class so that the main
method delegates most of its work to additional static methods that you define.
These methods should help organise the menu logic and improve the readability and
structure of your program.

Marking
Each project section is marked out of 10 (BookRecord, RecordManager, FileHandler and
Main).
All group members receive the average of the section marks, unless a member has not
made an adequate contribution to the project.
If a member’s section is marked significantly lower because it is incomplete, incorrect, or
missing required features, that mark will:
•
•
not be included in the group average, and
that student will receive this individual mark instead.
This ensures strong contributions are recognised, under-performance is treated fairly, and
all students remain accountable for their own section.
You are still encouraged to collaborate and discuss ideas with your group members as you
work on your individual sections.
Submission
Your group should submit a zip file containing the following files: BookRecord.java,
RecordManager.java, FileHandler.java and Main.java. (If you have 3 members you do not
need to submit Main.java). Only one submission is required for the group.
The deadline is 17:00 on Monday 6th April 2026.
